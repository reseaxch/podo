# Documentation workstream

Own durable product, architecture, and collaboration context.

- Keep `MVP_PLAN.md`, `USE_CASES.md`, and `WORKSTREAMS.md` mutually consistent.
- Prefer links to owning module READMEs over hand-maintained file inventories.
- Distinguish implemented behavior from planned behavior.
- Preserve safety requirements, approval gates, and measurable acceptance criteria.
- Update directly affected producer and consumer docs when a public contract changes.
- Do not document secrets, local credentials, or machine-specific private paths.

Validate links, commands, path names, and claims against the current repository before handoff.
