# Product and architecture documents

This directory stores durable project context:

- `INTEGRATION_GUIDE.md` is the operator-facing path for connecting a real
  repository, telemetry, Codex, and GitHub to the current local POC;
- `MVP_PLAN.md` defines scope, architecture, safety boundaries, and success criteria;
- `USE_CASES.md` defines user-visible flows and acceptance behavior;
- `WORKSTREAMS.md` maps module ownership and cross-team coordination.

Implementation details belong in code and module READMEs. Update these documents when product scope, architecture, contracts, operations, or ownership materially changes.
